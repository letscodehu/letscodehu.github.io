## Commands

```bash
npm run dev        # Start Vite dev server
npm run build      # Type-check, build static site, generate sitemap
npm run preview    # Preview production build locally
```

The build command runs: `vue-tsc -b && NODE_OPTIONS='--experimental-require-module' vite-ssg build && tsx scripts/generate-sitemap.ts`

There is no test suite or linter configured.

## Architecture

**letscode.hu** is a Vue 3 + TypeScript static site (SSG) deployed to GitHub Pages via `vite-ssg`. It's a bilingual (EN/HU) personal brand/training site.

### Routing & i18n

All routes are prefixed by language (`/en/...`, `/hu/...`). Language is detected from the browser, stored in `localStorage`, and handled in `src/router/`. Translations live in `src/i18n/en.json` and `src/i18n/hu.json`. The `useI18n()` composable provides translation lookups throughout the app.

### Static Site Generation

`vite-ssg` prerenders pages at build time. The list of prerendered paths is maintained in `src/seo/prerender-paths.ts`. Any new page route must be added there to appear in the built output. The sitemap is generated by `scripts/generate-sitemap.ts` after the build.

### Content

- **Blog posts**: Defined in `src/data/blog-posts.ts` (manifest) with markdown files in `content/`. Rendered via `marked`.
- **Case studies**: Defined in `src/data/case-studies.ts`.
- Both content types need entries in their data file and a new prerender path for SSG to work.

### Dark Mode

Detected from system preference and togglable by the user. State is persisted in both `localStorage` and a cookie. The initial detection script is inlined in `index.html` (before CSS loads) to avoid flash of wrong theme.

### Analytics & Consent

GA4, Google Ads, Facebook Pixel, LinkedIn, and Hotjar are loaded conditionally on cookie consent. All tracking is wired to the cookie consent banner — do not initialize trackers outside that flow.

### API

Two AWS Lambda endpoints (configured in `src/config.ts`) handle waitlist signup and email capture from the frontend. Endpoints differ between dev and prod environments.

### Key Files

| File | Purpose |
|------|---------|
| `src/main.ts` | App entry point, ViteSSG setup |
| `src/router/index.ts` | Route definitions with language prefixes |
| `src/i18n/en.json`, `hu.json` | All UI strings |
| `src/data/blog-posts.ts` | Blog post manifest |
| `src/seo/prerender-paths.ts` | Paths SSG must prerender |
| `src/config.ts` | API endpoint URLs |
| `src/site.ts` | Site origin and OG image config |
